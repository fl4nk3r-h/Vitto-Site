# Vitto Website Sitemap & Page Strategy

## 1. Homepage (`/`)

- **Why:** Establishes first impression, communicates Vitto’s value, and guides users to relevant solutions.
- **Audience:** CTO, C-suite, Product Owners, Decision Makers
- **Conversion Goal:** Encourage exploration, direct to platform/solutions, initiate demo interest.

## 2. AI-First Platform Page (`/platform`)

- **Why:** Explains the modular, AI-first architecture and unique modules.
- **Audience:** CTO, Product Managers, Solution Architects
- **Conversion Goal:** Book a demo, learn about modules, download docs.

## 3. Lending Lifecycle / Full Stack Automation Page (`/automation`)

- **Why:** Details Vitto’s automation capabilities across the lending lifecycle.
- **Audience:** Operations Lead, CTO, Transformation Heads
- **Conversion Goal:** Book a demo, read platform documentation.

## 4. Collections Intelligence Page (`/collections`)

- **Why:** Showcases advanced collections and recovery intelligence.
- **Audience:** Credit Risk Head, Collections Lead, CFO
- **Conversion Goal:** Book a demo, request more information.

## 5. Agentic AI Page (`/agentic-ai`)

- **Why:** Presents pre-built, autonomous AI agents for banking.
- **Audience:** CTO, AI/ML Lead, Product Owners
- **Conversion Goal:** Book a demo, read technical docs.

## 6. API Infrastructure Page (Recommended: `/api` or `/infrastructure`)

- **Why:** Explains API-first integration, developer experience, and security.
- **Audience:** CTO, Developers, Integration Leads
- **Conversion Goal:** Read API docs, start integration, contact sales.
- **Status:** Not found as a dedicated route. Consider adding for developer-focused conversions.

## 7. About / Why Vitto (`/about`)

- **Why:** Builds trust, shares mission, vision, and differentiators.
- **Audience:** All stakeholders, especially C-suite
- **Conversion Goal:** Build trust, contact, book demo.

## 8. Contact / Request Demo Page (`/contact`)

- **Why:** Provides a direct channel for demo requests and inquiries.
- **Audience:** Decision Makers, Prospects
- **Conversion Goal:** Submit contact/demo request.

## 9. Self Sign-Up Page (`/signup`)

- **Why:** Enables self-serve onboarding for new institutions.
- **Audience:** Operations Lead, Product Owner
- **Conversion Goal:** Complete sign-up, start onboarding.

---

> **Note:** All required pages except a dedicated “API Infrastructure” page are present. If API infrastructure is a key differentiator, consider adding `/api` or `/infrastructure` as a new route for developer and integration lead conversions.
